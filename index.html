  • ¶

    • twitter.js — the main file, does nothing more than handle options, initialise the main object, and bring everything together.

    • defaults.js — has both the options' defaults and the endpoint URLs. Also parses the package.json and provides version and package metadata.

    • utils.js — various utilities.

    • errors.js — custom errors for this library.

    • rest.js — the REST API. Handles request scheduling and rate limiting transparently, as well as taking care of making Twitter's configuration (URL lengths and media sizes) available.

    • parser.js — streaming Twitter JSON parser.
